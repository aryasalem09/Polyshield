## 2025-03-09 - [Path Traversal & Sandbox Escape via Weak Argument Classification and Flawed Relative Path Checking]
**Vulnerability:** A critical path traversal / sandbox escape existed in the filesystem scope enforcement (`src/sandbox.js`). The `inside` function blocked valid child directories prefixed with `..` (like `..secret`) due to `rel.startsWith("..")`. Meanwhile, `looksLikePath` only checked arguments matching a specific regex (`PATHISH_KEY`), allowing attackers to supply absolute paths or symlink escapes under keys not matching the regex (e.g., `arg: "link/passwd"`), bypassing scope checks entirely.
**Learning:** Checking for relative paths using a simple string `startsWith("..")` is unsafe and blocks valid file paths that just happen to begin with those characters. Furthermore, classifying paths solely by their key name rather than their value's structure is insufficient; any string containing a path separator could act as a path injection vector and bypass validation.
**Prevention:** Always compare `..` safely using path separators (e.g., `rel === ".." || rel.startsWith(".." + sep)`). When classifying strings dynamically as paths, incorporate checks for path separators (`/` or `\`) directly on the value, rather than only depending on arbitrary regex checks on the property key name.
