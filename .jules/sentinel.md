## 2024-05-15 - [CRITICAL] Fix Recursive Validation Bypass on deep object trees
**Vulnerability:** The sandbox path validation in `assertPathArgsInScope` and `remapArgs` traversed JSON objects recursively up to a depth of 8 but silently returned without validating deeper objects. An attacker could bypass path scope restrictions and access arbitrary files on the host system by nesting an escape attempt inside an object deeper than 8 levels (e.g., `{"a":{"b":{"c":{"d":{"e":{"f":{"g":{"h":{"path":"../escape.txt"}}}}}}}}}}`).
**Learning:** Security checks that iterate or traverse arbitrary tree structures must fail closed. When a hard limit (like recursion depth) is hit, returning silently means the data past the limit is accepted without validation. In this case, the depth boundary check failed open rather than failing closed.
**Prevention:** In any security validation logic where depth or size limits are applied to input, ensure the limit check throws an explicit error or halts the entire operation, instead of ignoring the unvalidated portions.
