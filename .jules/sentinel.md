## 2023-10-27 - Fail Closed on Depth Limit in Sandbox Traversal
**Vulnerability:** A critical sandbox bypass existed where path scoping and argument remapping failed to validate paths nested more than 8 levels deep in an argument tree. An attacker could pass a deeply nested object like `{ a: { b: ... { path: "../escape.txt" } ... } }` and the `assertPathArgsInScope` function would silently return instead of validating the path, allowing path traversal outside the scoped directory.
**Learning:** The recursion safety check `if (depth > 8) return;` incorrectly failed open, returning a success/continue signal rather than halting execution when the limits were exceeded. In a security validation context, bailing out early without enforcing the constraint means the constraint is effectively bypassed for complex inputs.
**Prevention:** Always fail closed on security constraints. If an input is too complex to parse or validate safely (e.g., exceeds recursion depth or size limits), explicitly throw an error and deny the action, rather than silently ignoring the validation step.
